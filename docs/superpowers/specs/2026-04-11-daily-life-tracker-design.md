# 每日记录网站 — 设计说明

**日期：** 2026-04-11  
**状态：** 已评审（对话确认）  
**范围：** 单用户、本机浏览器、无后端账号

---

## 1. 目标与成功标准

### 1.1 产品目标

支持个人记录 **工作、生活、锻炼、学习** 等日常行为：以 **习惯清单**（开关型 / 数值型）为主，辅以 **当日回顾与明日改进** 文字；以 **自然周（周一至周日）** 为单元做 **自动汇总** 与 **手动周总分 + 周回顾 + 下周计划**；并提供 **统计页** 浏览各周分数与进入周详情。

### 1.2 成功标准（验收）

- 用户在 **单设备浏览器** 内完成闭环：**配置习惯 → 每日打卡与文字 → 每周查看汇总并填写周分与计划 → 在统计页浏览各周**。
- 数据持久化在 **IndexedDB**，同浏览器配置下 **刷新与重开** 后数据仍在。
- 周界为 **本地时区** 下 **周一至周日**；界面展示周标签为 **`wk` + 年内周序号**（如 `wk02`）并附 **起止日期**；内部存储可使用稳定字符串键（如 ISO 周 `YYYY-Www`）以便排序与去重，实现阶段在计划中细化。
- **导出 / 导入备份** 不作为首版必交付；若有余力可作为后续迭代（例如「下载 JSON」）。

### 1.3 明确不在范围

- 多用户、登录、服务端同步。
- 跨设备实时同步。
- 换时区旅行时的记录迁移（首版按设备本地「日」与「周」计算，文档化已知限制即可）。

---

## 2. 架构方案（已定：方案二）

### 2.1 技术选型

- **形态：** 纯前端 **SPA**（例如 **Vite + React + TypeScript**）。
- **存储：** **IndexedDB**（建议通过 Dexie 等轻量封装访问）。
- **部署：** 可构建为 **静态资源**，无自有后端依赖。

### 2.2 理由摘要

相较 LocalStorage 单文件 JSON，IndexedDB 更适合习惯、日记录、周记录的分表与按周聚合查询；相较 Electron/Tauri，更贴合「仅本机浏览器、单用户」的体量与维护成本。

## 3. 信息架构与界面

### 3.1 核心实体

| 实体 | 说明 |
|------|------|
| **习惯** | 名称、类型（**开关型** / **数值型**）、可选单位文案（如 km、小时）、排序、是否启用；删除采用 **软删除（归档）**，历史日记录保留可解析的名称或快照。 |
| **日记录** | 以 **日历日** 为键（`YYYY-MM-DD`，本地日界）。含各习惯当日值（开关为 boolean；数值为 number，未填与 0 的语义在实现中统一：数值型「未填」用 null/缺失表示，避免与真实 0 混淆）、**今日回顾**、**明日改进**（纯文本）。 |
| **周记录** | 对应 **周一至周日** 的一周。含 **周总分（手动）**、**周回顾**、**下周计划**（文本）；与 7 天日记录 **解耦**（允许仅有周总结而无日打卡）。 |

### 3.2 主要页面 / 视图

1. **今日 / 选日**：习惯打卡 + 两段日文字。
2. **习惯管理**：增删改、排序、类型与单位；归档后不出现在默认打卡列表。
3. **本周**：上方 **自动汇总**（见第 4 节），下方 **周总分 + 周回顾 + 下周计划**。
4. **统计**：周列表（周标签、日期范围、周总分）；可导航至对应 **本周** 视图。

导航可采用顶栏或侧栏：**今日｜本周｜统计｜习惯**（文案可在实现中微调）。

## 4. 数据流与周汇总规则

### 4.1 写入与读取

- **习惯**：变更后立即持久化。
- **日记录**：用户编辑习惯值与文字；推荐 **防抖自动保存**；当日首次写入时创建该日记录。
- **周记录**：在「本周」或从统计进入的某周编辑周字段，独立持久化。
- **统计页**：只读；聚合日数据生成汇总，不反向修改日记录。

### 4.2 周范围

- 使用用户 **设备本地时区**，从 **周一 00:00** 到 **周日 24:00**（即周日结束后的日界切换），以 **`YYYY-MM-DD`** 作为「日」唯一键，减少夏令时边界问题。

### 4.3 自动汇总（首版）

对每周内、与习惯相关的数据：

| 习惯类型 | 规则 |
|----------|------|
| **开关型** | **完成天数**：该周内「存在日记录且标记为已完成」的天数。**未打开应用、无日记录** 的天数 **不计入「未完成」**；界面可辅助展示「本周共 7 天，其中 N 天未记录」类说明（实现可选）。 |
| **数值型** | **本周合计**；**有填写数值的天数**；**日均** = 合计 ÷ 有值天数（有值天数为 0 时不展示日均）。数值型「未记」与「记 0」须在实现层明确区分（推荐未记为 null，不进入合计与有值天数）。 |

### 4.4 习惯归档与历史

- **归档（软删除）**：习惯不再出现在默认列表与今日打卡；**历史日记录**仍关联原习惯 id，并具备可展示名称（名称快照或可从归档记录解析），避免硬删导致数据无法解释。

## 5. 错误处理与降级

- **IndexedDB 写入失败或空间不足**：非阻塞 **提示**（如 Toast），建议稍后重试；导出能力上线后再引导备份。
- **读取失败或 schema 不兼容**：模块级友好错误说明，不静默吞掉；使用 **schema 版本号**，升级策略在实现计划中定义（首版可从 v1 起）。

---

## 6. 测试范围与验收（工程）

### 6.1 建议自动化

- **日期 / 周界工具**：任意日期 → 正确的一周区间、周展示标签与排序键（含跨月、跨年周）。
- **周汇总逻辑**：给定固定模拟日数据，开关完成天数、数值合计 / 有值天数 / 日均与第 4.3 节一致。

### 6.2 可选

- **E2E**（如 Playwright）：习惯 → 日记 → 本周汇总 → 周分 → 统计；非首版阻塞条件。

### 6.3 发布前手测清单

1. 新建开关型与数值型习惯，今日填写后刷新仍在。
2. 历史某日补记后，本周汇总随之更新。
3. 「本周」自动区与手填周分、周回顾、下周计划保存后，从统计进入该周一致。
4. 归档习惯后，历史日记录仍可理解、无未处理异常。
5. 无习惯、无记录时各页有空状态说明，不白屏。
